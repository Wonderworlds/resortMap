# Story 2.1: builder-core ŌĆö Authoring Operations & Serialization

Status: review

## Story

As a builder-react adapter,
I want builder-core to expose immutable operations for composing a `MapConfig` (create, add/remove/update POIs, add/remove nodes and edges) plus serialize/parse round-trip functions,
So that the UI can call pure functions to mutate map state and export a valid `.gwmap` file with guaranteed schema compliance.

## Acceptance Criteria

1. **Given** a valid `MapMeta` (`backgroundImageUrl`, `center`, `scale`) **When** I call `createMapConfig(meta)` **Then** it returns a `MapConfig` with `version: "1.0"`, `pois: []`, and `graph: { nodes: [], edges: [] }`

2. **Given** an existing `MapConfig` **When** I call `addPoi(config, { label, position, tags })` **Then** it returns a new `MapConfig` with the POI appended and `id` generated by `crypto.randomUUID()` **And** the original `config` is unchanged

3. **Given** a `MapConfig` with at least one POI **When** I call `removePoi(config, poiId)` **Then** it returns a new `MapConfig` with that POI removed

4. **Given** a `MapConfig` with at least one POI **When** I call `updatePoi(config, poiId, { label: "New Name" })` **Then** it returns a new `MapConfig` where only the patched fields change and all others are preserved

5. **Given** a `MapConfig` **When** I call `addNode(config, { position })` **Then** it returns a new `MapConfig` with the node appended and `id` from `crypto.randomUUID()`

6. **Given** a `MapConfig` with two nodes `A` and `B` **When** I call `addEdge(config, { from: A.id, to: B.id })` **Then** it returns a new `MapConfig` with the edge appended

7. **Given** a `MapConfig` with node `N` referenced by one or more edges **When** I call `removeNode(config, N.id)` **Then** it returns a new `MapConfig` with the node removed **and** all edges where `from` or `to` equals `N.id` removed

8. **Given** a `MapConfig` with an edge from node `A` to node `B` **When** I call `removeEdge(config, A.id, B.id)` **Then** it returns a new `MapConfig` with that specific edge removed **And** both nodes remain in the graph **And** the original `config` is unchanged

9. **Given** any valid `MapConfig` **When** I call `serializeGwmap(config)` then `parseGwmap(result)` **Then** the parsed result deep-equals the original `config` (lossless round-trip, FR1)

10. **Given** a malformed JSON string **When** I call `parseGwmap(raw)` **Then** it throws with `code: ErrorCode.GWMAP_PARSE_ERROR`

11. **Given** builder-core's source files **When** I grep for `import.*react` **Then** zero matches are found (NFR4)

### `mapConfig.ts` ŌĆö Complete Implementation

```ts
import type { MapConfig, MapMeta, POI, GraphNode, GraphEdge } from '@resort-map/types';
import { generateId } from './utils/idGeneration.ts';

export function createMapConfig(meta: MapMeta): MapConfig {
  return {
    version: '1.0',
    map: meta,
    pois: [],
    graph: { nodes: [], edges: [] },
  };
}

export function addPoi(config: MapConfig, poi: Omit<POI, 'id'>): MapConfig {
  const newPoi: POI = { ...poi, id: generateId() };
  return { ...config, pois: [...config.pois, newPoi] };
}

export function removePoi(config: MapConfig, poiId: string): MapConfig {
  return { ...config, pois: config.pois.filter(p => p.id !== poiId) };
}

export function updatePoi(
  config: MapConfig,
  poiId: string,
  patch: Partial<Omit<POI, 'id'>>,
): MapConfig {
  return {
    ...config,
    pois: config.pois.map(p => (p.id === poiId ? { ...p, ...patch } : p)),
  };
}

export function addNode(config: MapConfig, node: Omit<GraphNode, 'id'>): MapConfig {
  const newNode: GraphNode = { ...node, id: generateId() };
  return {
    ...config,
    graph: { ...config.graph, nodes: [...config.graph.nodes, newNode] },
  };
}

export function addEdge(config: MapConfig, edge: GraphEdge): MapConfig {
  return {
    ...config,
    graph: { ...config.graph, edges: [...config.graph.edges, edge] },
  };
}

export function removeNode(config: MapConfig, nodeId: string): MapConfig {
  return {
    ...config,
    graph: {
      nodes: config.graph.nodes.filter(n => n.id !== nodeId),
      edges: config.graph.edges.filter(e => e.from !== nodeId && e.to !== nodeId),
    },
  };
}

export function removeEdge(config: MapConfig, from: string, to: string): MapConfig {
  return {
    ...config,
    graph: {
      ...config.graph,
      edges: config.graph.edges.filter(e => !(e.from === from && e.to === to)),
    },
  };
}
```

### `serialization.ts` ŌĆö Complete Implementation

```ts
import type { MapConfig } from '@resort-map/types';
import { validateGwmap, ErrorCode } from '@resort-map/types';

export function serializeGwmap(config: MapConfig): string {
  return JSON.stringify(config);
}

export function parseGwmap(raw: string): MapConfig {
  let parsed: unknown;
  try {
    parsed = JSON.parse(raw);
  } catch {
    throw Object.assign(
      new Error('Failed to parse .gwmap: invalid JSON'),
      { code: ErrorCode.GWMAP_PARSE_ERROR },
    );
  }
  return validateGwmap(parsed);
}
```

> `validateGwmap` already throws with the correct `ErrorCode` (`GWMAP_PARSE_ERROR` or `GWMAP_VERSION_MISMATCH`) for schema errors, so those exceptions propagate naturally ŌĆö no re-wrapping needed.

### Architecture Compliance (MANDATORY)

1. **`verbatimModuleSyntax: true`** ŌĆö ALL type imports MUST use `import type`:
   ```ts
   import type { MapConfig, MapMeta, POI, GraphNode, GraphEdge } from '@resort-map/types';
   import { validateGwmap, ErrorCode } from '@resort-map/types'; // values: no 'type'
   ```
   Using `import { POI }` for a type will cause TypeScript error TS1484. This is a compile-time failure.

2. **`noUncheckedIndexedAccess: true`** ŌĆö Array subscript access returns `T | undefined`:
   ```ts
   // WRONG ŌĆö TypeScript error: 'result.pois[0]' is possibly 'undefined'
   expect(result.pois[0].label).toBe('Restaurant');

   // CORRECT ŌĆö use non-null assertion when you know the array is non-empty
   expect(result.pois[0]!.label).toBe('Restaurant');
   ```
   In test files, always do `expect(result.pois).toHaveLength(n)` BEFORE accessing by index, then use `!`.

3. **Zero React imports** ŌĆö `grep -r 'import.*react' packages/builder-core/src/` must return no results (NFR4, AC 11).

4. **Immutability** ŌĆö Every function returns a NEW object. Never mutate a parameter:
   ```ts
   // WRONG
   config.pois.push(newPoi);
   return config;

   // CORRECT
   return { ...config, pois: [...config.pois, newPoi] };
   ```

5. **`crypto.randomUUID()` for IDs** ŌĆö `crypto` is a global in Bun, Node 18+, and browsers. No import needed (ADR-007).

6. **Error throw pattern** (ADR-004) ŌĆö in `parseGwmap`:
   ```ts
   throw Object.assign(
     new Error('Failed to parse .gwmap: invalid JSON'),
     { code: ErrorCode.GWMAP_PARSE_ERROR },
   );
   ```

7. **Explicit return types** on all exported functions (mandatory per architecture).

8. **Import path convention** ŌĆö use relative paths with `.ts` extension within the package:
   ```ts
   import { generateId } from './utils/idGeneration.ts';
   ```

9. **Public API boundary** ŌĆö only `src/index.ts` exports public symbols. `mapConfig.ts`, `serialization.ts`, and `utils/idGeneration.ts` are implementation files ŌĆö other packages must import from `@resort-map/builder-core`, never from deep paths.

### Previous Story Learnings (Stories 1.1 ŌĆō 1.3)

- **`verbatimModuleSyntax: true` causes compile failures for type-value confusion** ŌĆö always check if an import is a type or a value. `validateGwmap` is a value (function), `ErrorCode` is a value (const object); `MapConfig`, `POI`, etc. are types only.
- **`noUncheckedIndexedAccess: true` affects test files** ŌĆö accessing `arr[0]` returns `T | undefined`. Always use `arr[0]!` or a `toBeDefined()` check first. Forgetting this leads to TypeScript errors in tests.
- **`bun test` auto-discovers `*.test.ts` recursively** ŌĆö no configuration needed. Tests in `src/__tests__/` are found automatically.
- **Within-package imports need `.ts` extension** ŌĆö `import { generateId } from './utils/idGeneration.ts'` (with `.ts`), not `'.utils/idGeneration'`.
- **JSON imports from fixture package work without assertion** ŌĆö `import complexMap from '@resort-map/types/fixtures/complex.gwmap.json'` works natively in Bun. No `assert { type: 'json' }` needed.
- **`bun test` runs from workspace root** ŌĆö the command is `bun test` at the workspace root, which discovers tests across ALL packages.

### `parseGwmap` Error-Handling Design

`parseGwmap` in builder-core is a thin wrapper that:
1. Catches `JSON.parse` `SyntaxError` and converts it to `GWMAP_PARSE_ERROR`
2. Delegates all schema validation to `validateGwmap` from `@resort-map/types`

This means:
- Malformed JSON ŌåÆ caught by `try/catch`, thrown as `GWMAP_PARSE_ERROR`
- Valid JSON, bad schema ŌåÆ `validateGwmap` throws `GWMAP_PARSE_ERROR` (propagates naturally)
- Valid JSON, unknown major version ŌåÆ `validateGwmap` throws `GWMAP_VERSION_MISMATCH` (propagates naturally)

Do NOT re-wrap errors that come from `validateGwmap` ŌĆö let them propagate.

### `removeEdge` Matching Semantics

`removeEdge(config, from, to)` removes ALL edges where `edge.from === from && edge.to === to`. In practice, a well-formed graph has at most one edge per directional pair, so this removes exactly one edge. The implementation uses `.filter()`:
```ts
edges: config.graph.edges.filter(e => !(e.from === from && e.to === to))
```

### `updatePoi` Patch Semantics

`updatePoi` uses object spread `{ ...poi, ...patch }`. This means:
- Fields present in `patch` replace the POI's value
- Fields absent from `patch` are preserved unchanged
- The POI's `id` cannot be changed (parameter type is `Partial<Omit<POI, 'id'>>`)
- Setting a field to `undefined` in the patch would technically remove optional fields ŌĆö the tests don't cover this edge case and it need not be explicitly handled
