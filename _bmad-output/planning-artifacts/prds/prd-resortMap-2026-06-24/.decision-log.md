# Decision Log — ResortMap Builder UI/UX Redesign (Epic 7)

Created: 2026-06-24
Author: Floran (facilitated via bmad-prd)

---

## D-1: Preview mode scope (2026-06-24)
**Decision:** Preview mode renders `@resort-map/viewer` with `preview={true}` prop. This prop disables filtering, routing, and all interactive features beyond map+POI display.
**Source:** User answer Q1.
**Implication:** Viewer package must support a `preview` boolean prop; if not present today, it is a dependency of this epic.

## D-2: Map ↔ Content list sync (2026-06-24)
**Decision:** Hovering a POI in the Content tab highlights it on the map. Clicking a POI on the map scrolls the Content list to that item — *only* when the Content tab is active. When Content tab is active, Select tool is always forced.
**Source:** User answer Q2.

## D-3: onImport hook (2026-06-24)
**Decision:** The onImport hook (recently implemented in MapMetaPanel) is out of scope for this PRD. It is already in the codebase and is not redesigned here.
**Source:** User answer Q3.

## D-4: Theme system (2026-06-24)
**Decision:** Default palette derived from reference image (teal primary, off-white sidebars, light gray canvas). A `ThemeConfig` prop on App exposes the full palette for override. Implementation via MUI ThemeProvider.
**Source:** User answer Q4.

## D-5: Responsiveness (2026-06-24)
**Decision:** Builder must be "a little bit responsive" — panels remain usable at standard laptop widths (≥1024px). No requirement for mobile-first layout or full breakpoint system.
**Source:** User answer Q5.

## D-6: Working mode (2026-06-24)
**Decision:** Fast path PRD. `[ASSUMPTION]` tags used for inferred details.
**Source:** Context volume and user pace.

## D-7: Viewer `preview` prop (2026-06-24)
**Decision:** `@resort-map/viewer` does not currently accept a `preview` prop. Implementing it is a prerequisite story in this epic (viewer-side change). The builder's Preview mode story depends on it.
**Source:** OQ-1 resolved by Floran.

## D-8: Save hook (2026-06-24)
**Decision:** Save action does not download by default. Instead, `<App>` accepts an `onSave?: (config: MapConfig) => void` hook prop. The caller decides what to do (download, POST to server, etc.). If no hook is provided, a default download behavior is used as fallback.
**Source:** OQ-2 resolved by Floran.
**Implication:** `MapConfig` and the hook type are exported from `packages/builder-react`.

## D-9: App name in Top Nav Bar (2026-06-24)
**Decision:** "ResortMap Builder" as plain text (MUI Typography).
**Source:** OQ-3 resolved by Floran.

## D-10: ThemeConfig type (2026-06-24)
**Decision:** `ThemeConfig = Partial<ThemeOptions>` from MUI — full options, re-exported with a friendly alias. Documented example provided in package.
**Source:** OQ-4 resolved by Floran.
